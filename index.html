The Business Model Canvas

Key Partners

We are our own suppliers. Our key partner would be google due to the sdk for browser extensions we will be creating our product out of. Google will pretty much be doing all the stability that relates between the user and our app.

Key Activities

-A base website for trust as well as any error reporting
-The platform would be a chrome extension so it would be entirely client sided

Value Proposition

The price would be minimal. Likely the only foreseeable price would be the one to run a home website.
Customization would most likely inclue automatic keying and whether it would be displayed or not for the user to see.

Customer Relationship

Customer relations would almost entirely be non existent after download. It's client side security program so other than error and bug reporting any contact would be minimal. A main website would most likely be necessary for those few needs

Customer Segments

People who want more security when they send things out onto the internet. They would most likely be people with a slight to extreme paranoia of government or other foreign listeners

Key Resources

it would need to use
-Chrome extension sdk
-computers
- a series of security algorithms
-probably will also require a server to run a home website off of

Channels

Open sourced websites and by forum announcement would probably be the best outlets to get the product out. Those people tend to be more technical and more geared to wanting extra security

Cost Structure

The cost structure would mostly just include the time invested. A website server probably.

Revenue Streams

Currently the product does not actually exist. They would most liekly not have to pay anything from my understanding. Our group hasn't talked about a price so I'm expecting free software.
